# 커뮤니티/블로그 → 텔레그램 자동 전송

## 개요
특정 커뮤니티 게시판/블로그의 새 글을 자동 감지하여 텔레그램으로 전송하는 자동화 시스템

## 리포지토리
https://github.com/edward071007-hash/ddanzi-telegram (public)

## 구성
```
ddanzi-telegram/
├── config.json              ← 감시 대상 관리 (여기만 수정)
├── ddanzi_to_telegram.py    ← 딴지일보 크롤러
├── naver_to_telegram.py     ← 네이버 블로그 크롤러 (RSS)
├── .github/workflows/
│   ├── ddanzi_check.yml     ← 06~08시, 16~18시 / 10분 간격
│   └── naver_check.yml      ← 30분 간격 (24시간)
├── ddanzi_last_ids.json     ← 딴지 마지막 전송 글 ID (자동 커밋)
├── naver_last_ids.json      ← 네이버 마지막 전송 글 ID (자동 커밋)
├── .gitignore
└── 작업기록.md
```

## 감시 대상 (config.json)
- **딴지일보:** 정대만mitsui (증시 요약)
- **네이버 블로그:** onejejuwave — 제주바람의 투자기록

## 텔레그램 봇 정보
- 봇 토큰: GitHub Secrets에 `TELEGRAM_BOT_TOKEN`으로 저장
- Chat ID: GitHub Secrets에 `TELEGRAM_CHAT_ID`으로 저장 (1968767262)

## 실행 스케줄 (한국시간 기준)
- **딴지일보:** 06:00~08:00, 16:00~18:00 — 10분 간격
- **네이버 블로그:** 24시간 — 30분 간격

## 추가/삭제 방법
- **추가:** config.json에 ID 추가 → 첫 실행에서 최신 글 ID만 저장 (기존 글 전송 없음), 이후 새 글만 전송
- **삭제:** config.json에서 ID 제거 → 즉시 중단
- 새 커뮤니티(딴지/네이버 외 다른 사이트) 추가 시: 크롤러 스크립트 1개 + yml 1개 추가 필요

## 참고
- 딴지일보: 닉네임 검색 → 글 작성자 CSS 클래스로 정확히 필터링
- 네이버 블로그: RSS 피드 기반, 블로그 ID만 있으면 동작
- last_ids.json은 워크플로우가 리포에 자동 커밋 (캐시 만료 걱정 없음)

## 비용
- 전부 무료 (GitHub Actions public 리포 무제한)
